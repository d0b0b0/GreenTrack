# 🗄️ Подключение бесплатной базы данных (Supabase)

GreenTrack работает **сразу** в demo-режиме (данные в браузере). Чтобы включить
настоящую бесплатную облачную базу данных с регистрацией, синхронизацией между
устройствами и общим рейтингом — подключите **Supabase** (бесплатный тариф).

> ⏱️ Занимает ~5 минут. Кредитная карта не нужна.

---

## Шаг 1. Создайте проект Supabase

1. Зайдите на **https://supabase.com** → **Start your project** → войдите через GitHub.
2. Нажмите **New project**.
3. Заполните:
   - **Name**: `greentrack`
   - **Database Password**: придумайте надёжный пароль (сохраните его).
   - **Region**: ближайший к вам (например, `Central EU (Frankfurt)`).
4. Нажмите **Create new project** и подождите ~1–2 минуты, пока проект запустится.

## Шаг 2. Создайте таблицы

1. В проекте откройте слева **SQL Editor** → **New query**.
2. Откройте файл [`supabase/schema.sql`](./supabase/schema.sql) из этого репозитория,
   скопируйте всё его содержимое в редактор.
3. Нажмите **Run** (или `Ctrl+Enter`). Должно появиться `Success. No rows returned`.

Это создаст таблицы `profiles`, `activities`, `user_challenges`,
`user_achievements`, настроит безопасность (RLS) и триггер автосоздания профиля.

## Шаг 3. Отключите подтверждение email (для удобной регистрации)

1. Слева: **Authentication** → **Sign In / Providers** → **Email**.
2. Выключите **Confirm email** (Disable "Confirm email"). Сохраните.

> Без этого пользователю нужно подтверждать почту перед первым входом. Для
> студенческого проекта удобнее выключить.

## Шаг 4. Скопируйте ключи

1. Слева: **Project Settings** (⚙️) → **API**.
2. Скопируйте два значения:
   - **Project URL** → это `VITE_SUPABASE_URL`
   - **Project API keys → `anon` `public`** → это `VITE_SUPABASE_ANON_KEY`

## Шаг 5. Добавьте ключи в проект

### Локально

Создайте файл `.env` в корне проекта и вставьте:

```
VITE_SUPABASE_URL=https://xxxxxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOi....(длинный ключ)
```

Перезапустите дев-сервер: `npm run dev`. В приложении исчезнет жёлтый баннер
«Demo-режим» — значит, облачная БД подключена.

### На Vercel

В настройках проекта Vercel: **Settings → Environment Variables** добавьте те же
две переменные (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`) и сделайте
**Redeploy**.

---

## Готово 🎉

Теперь регистрация, вход, активности, челленджи, достижения и рейтинг
сохраняются в облачной базе и доступны с любого устройства.

### Частые вопросы

- **Безопасно ли публиковать `anon`-ключ?** Да. Это публичный ключ; доступ к
  данным ограничивает Row Level Security (настроена в `schema.sql`). Никогда не
  публикуйте `service_role`-ключ.
- **Email в рейтинге виден другим?** Нет. Email хранится в `auth.users` и не
  попадает в публичную таблицу `profiles`. В рейтинге видны только имя, аватар,
  уровень и баллы (и только если включён «Публичный профиль» в настройках).
- **Бесплатный тариф закончится?** Free tier Supabase щедрый (500 МБ БД,
  до 50k активных пользователей в месяц). Для студенческого проекта более чем
  достаточно. Проект может «засыпать» после недели без активности — просто зайдите.
